# 消息格式与协作管理

## @ 标签格式（严格遵守）

**正确格式**：
```
<at user_id="ou_xxxx">名字</at>
```

**常见错误**：

| 错误写法 | 问题 |
|---------|------|
| `@名字` | 明文 @ 不触发飞书投递，对方收不到 |
| `<at user_id="cli_xxx">` | cli_ 是会话 ID，不是 open_id，投递失败 |
| 一条消息多个 `<at>` | 每次最多 @ 1 个 Agent |

## 任务型 vs 通知型

### 任务型 @（需要对方回复）

直接使用 `<at>` 标签，对方需要 @ 回你：
```
<at user_id="ou_xxxx">司南</at> 请帮我调研一下 MoE 架构的最新进展。
完成后请 @ 回我汇报结果。
```

### 通知型 @（不需要对方回复）

加 🔕仅通知 标记：
```
🔕仅通知 <at user_id="ou_xxxx">司南</at> 之前的调研任务已取消，无需继续。
```

对方收到后不需要 @ 回你。

## 区分"提到"和"请求"

- **提到**：用文字名字 → "司南之前调研过这个话题"
- **请求**：用 `<at>` 标签 → `<at user_id="ou_xxx">司南</at> 请帮忙调研`

只有需要对方执行操作时，才用 `<at>` 标签。

## 中断与取消

### 识别中断信号

| 用户说 | 含义 |
|--------|------|
| "停"、"停止"、"取消" | 停止所有协作 |
| "算了"、"不用了" | 取消当前任务 |
| "方向错了"、"不是这样" | 需要调整方向 |
| "先等等"、"暂停" | 暂停，等待指示 |

### 中断处理

1. **确认** — 回复用户，简述当前进度
2. **通知** — 用 🔕仅通知 告知正在处理中的 Agent
3. **等待** — 如果后续收到已取消任务的回传，不再 @ 回，告知用户即可

```
收到，已停止协作。目前司南的调研已完成，灵犀的内容创作进行中。

🔕仅通知 <at user_id="ou_xxxx">灵犀</at> 之前分配的内容创作任务已取消，无需继续。

需要我做什么调整？
```

## 协作模式

| 模式 | 触发词 | 行为 |
|------|--------|------|
| 正常模式 | 默认 | 用户要求或关键字触发时 @ |
| 独立模式 | "只跟你聊"、"不要叫别人" | 不 @ 任何 bot |
| 指定模式 | "只找 xxx 和 yyy" | 只 @ 用户指定的 bot |
| 全力模式 | "全员协作"、"大家一起" | 主动拆解分配给所有相关 bot |

- 模式切换只影响当前对话，不是永久设置
- 被动协作（收到 @ ）不受模式限制
- 独立模式下遇到超出能力的任务，提示用户是否切回协作模式
